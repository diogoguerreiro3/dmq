## How to Contribute

### Reporting Bugs

If you find a bug, please open an [issue](https://github.com/your-username/dmq/issues) with:

- Clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Screenshots (if applicable)
- System information (OS, Python version, etc.)

### Suggesting Enhancements

Have ideas to improve DMQ? Great! Open an issue with:

- Clear description of the feature
- Why it would be useful
- Usage examples (if possible)

### Pull Requests

1. **Fork the repository**
2. **Create a branch** for your feature:
   ```bash
   git checkout -b feature/MyNewFeature
   ```
3. **Commit your changes**:
   ```bash
   git commit -m 'Add MyNewFeature'
   ```
4. **Push to the branch**:
   ```bash
   git push origin feature/MyNewFeature
   ```
5. **Open a Pull Request**

### Style Guide

- Use Python 3.10+
- Comment complex code
- Maintain consistency with existing code
- Test changes before submitting


### Commit Structure

Use descriptive commit messages:

```
[TYPE] short description

Longer description if necessary
```

Types:
- `FEAT`: New feature
- `BUG`: Bug fix
- `DOC`: Documentation changes
- `STYLE`: Formatting, missing semi-colons, etc
- `REFACTOR`: Code refactoring
- `TEST`: Adding tests
- `MAIN`: Maintenance

## Code of Conduct

Be respectful and constructive in interactions with other contributors.

## Questions?

If you have questions, open an issue or get in touch!
